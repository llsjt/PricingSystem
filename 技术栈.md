# 电商智能定价平台技术栈

## 1. 项目定位

这是一个电商智能定价平台，采用前后端分离 + 双后端协作架构：

- 前端负责页面展示、交互、任务发起和结果查看
- Java 后端负责对外 API、鉴权、业务编排、SSE 推送和数据读取
- Python 后端负责内部智能定价任务执行和多智能体编排
- MySQL 负责业务数据持久化
- RabbitMQ 负责异步任务派发

## 2. 总体架构

```text
Frontend (Vue 3 + TypeScript + Vite)
  -> Java Backend (Spring Boot 3.2, public API)
    -> Python Worker (FastAPI, internal API)
      -> CrewAI multi-agent pricing workflow
        -> MySQL 8.4
        -> RabbitMQ 3.13
```

## 6. AI 与多智能体相关技术

- CrewAI 用于多智能体编排
- 项目中包含分析、市场、风控、管理等多角色 Agent 协作
- 市场情报当前基于本地天猫 CSV 竞品索引
- 本地 CSV 会预先构建为 SQLite 竞品索引，Python 侧按二级类目、一级类目和商品标题关键词匹配样本；当前不代表实时平台行情接入。
- “应用价格建议”属于系统内业务闭环：Java 后端校验调价幅度和毛利率后更新本系统商品当前售价，并记录应用审计信息；当前不直接调用外部电商平台改价 API。
- Python 侧实现了 OpenAI-compatible Chat Completions 适配
- 用户在个人中心维护自己的模型地址、模型名与密钥；任务创建时会把该配置快照写入 `pricing_task`
- Python Worker 执行时只使用任务快照中的用户配置，不再读取系统级默认模型

## 8. 接口与通信方式

### 对外接口

- REST API
- SSE（Server-Sent Events）

### 内部通信

- Java -> Python 内部 HTTP 调用

### 关键协议与机制

- `/api/**` 统一前端访问入口
- `GET /api/pricing/tasks/{taskId}/events` 用于任务实时流
- `X-Internal-Token` 用于 Java 和 Python 间内部鉴权
- `Bearer Token` 用于前端访问鉴权
- Refresh Token Cookie 用于会话续期

## 10. 本地开发与脚本体系

- Windows PowerShell 启动脚本
- `scripts/start-local-dev.ps1` 用于一键启动前端、Java 后端、Python 后端
- 前端本地开发命令：`npm run dev`
- Java 本地开发命令：`mvn spring-boot:run`
- Python 本地开发命令：`python -m uvicorn app.main:app --reload --port 8000`
- Windows 兼容备用命令：`python run_server.py`

## 11. 技术栈一句话总结

本项目采用 `Vue 3 + TypeScript + Vite` 的前端方案，配合 `Spring Boot 3.2 + Java 21` 作为主业务后端，使用 `FastAPI + CrewAI + Python 3.12` 执行内部智能定价工作流，并以 `MySQL + RabbitMQ + Docker + Nginx` 完成数据存储、异步派发与部署运行。
